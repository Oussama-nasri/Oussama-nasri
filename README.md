## 📂 Featured Deep-Dives

### 1. 🩺 [ClinicalTruth — Multilingual Medical Verification Engine](https://github.com/Oussama-nasri/ClinicalTruth)
*A production-grade RAG system that verifies clinical claims against Arabic, French, and English medical guidelines — combining hybrid retrieval, NLI-based contradiction detection, and a multi-step agentic verification pipeline.*
*   **Engineering Highlights**:
    *   **Hybrid Retrieval**: Dense vector search (Qdrant + BGE-M3) fused with BM25 sparse retrieval — maximizing recall across both semantic and lexical signals.
    *   **NLI Contradiction Detection**: Entailment/contradiction classification pipeline that flags claims conflicting with retrieved guideline passages.
    *   **Arabic RTL Handling**: Preprocessing normalizes Arabic presentation forms and handles right-to-left document structure before chunking.
    *   **Agentic Verification Loop**: Multi-step agent that queries, retrieves, verifies, and synthesizes an evidence-backed verdict with source citations.
*   **Key Stack**: Python, FastAPI, Qdrant, Elasticsearch, BGE-M3, HuggingFace Transformers, Docker.

### 2. ⚙️ [OSS Dependency & API Compatibility Knowledge Graph](https://github.com/Oussama-nasri/oss-versionning-knowledge-graph)
*An ontology-backed knowledge graph modeling open-source dependency trees and API compatibility constraints — enabling automated breaking-change detection across package version histories.*
*   **Engineering Highlights**:
    *   **Graph Ontology Design**: Neo4j schema modeling packages, versions, APIs, and compatibility edges with typed relationships.
    *   **NLP Extraction Pipeline**: Extracts dependency declarations and API signatures from source code and changelogs via spaCy + custom NER.
    *   **Hybrid Similarity Scoring**: Combines rapidfuzz string matching with embedding similarity for fuzzy package name disambiguation.
    *   **FastAPI Query Interface**: REST API to traverse compatibility paths, detect breaking changes, and retrieve dependency sub-graphs.
*   **Key Stack**: Python, Neo4j, FastAPI, spaCy, rapidfuzz, sentence-transformers, Docker.

### 3. 🏭 [IntelliScrape — B2B Industrial Intelligence Platform](https://github.com/Oussama-nasri/IntelliScrape)
*A web intelligence platform that scrapes, structures, and enriches B2B industrial company data — powering lead generation and competitive market analysis workflows.*
*   **Engineering Highlights**:
    *   **Multi-Engine Scraping**: Playwright (dynamic JS rendering) + Selenium for legacy sites, with intelligent fallback routing between engines.
    *   **Structured Enrichment**: Extracted entities (company, products, contacts) normalized and indexed in Elasticsearch for full-text + filtered search.
    *   **Async Job Pipeline**: FastAPI + Celery + Redis task queue for concurrent scraping jobs with real-time status tracking.
    *   **LinkedIn Data Collection**: Automated LinkedIn profile scraping with rate-limit management and proxy rotation.
*   **Key Stack**: Python, Playwright, Selenium, Elasticsearch, FastAPI, Celery, Redis, Docker.

### 4. 🎙️ [voxtral-deployement — Dockerized Voxtral Speech Model](https://github.com/Oussama-nasri/voxtral-deployement)
*A production-ready Dockerized deployment of Mistral's Voxtral Mini 4B Realtime — exposing a WebSocket-based real-time transcription API with GPU acceleration and a FastAPI gateway layer.*
*   **Engineering Highlights**:
    *   **vLLM Backend**: Voxtral Mini 4B Realtime served via vLLM's OpenAI-compatible endpoint with Voxtral-specific WebSocket protocol handling.
    *   **WebSocket Streaming**: Real-time bidirectional audio streaming with chunk-based transcription delivery and low first-token latency.
    *   **GPU-Optimized Container**: Multi-stage Docker build with CUDA base, VRAM constraint management, and clean Compose service networking.
    *   **FastAPI Gateway**: Auth, rate limiting, and streaming proxy layer in front of the vLLM server — decoupling inference from the client API surface.
*   **Key Stack**: Python, vLLM, FastAPI, WebSockets, Docker, CUDA, Mistral Voxtral.

### 5. 📈 [investrog — Agentic Investment Research Assistant](https://github.com/Oussama-nasri/investrog)
*A personalized investment research agent that retrieves, synthesizes, and presents financial intelligence — combining real-time data retrieval with RAG-backed company analysis and a tool-calling architecture.*
*   **Engineering Highlights**:
    *   **MCP-Tooled Research Agent**: Modular tool-calling architecture where the agent selects retrieval, analysis, or summarization tools dynamically per query.
    *   **Personalized Portfolio Context**: User-level portfolio state drives contextual research prioritization and report focus.
    *   **Hybrid Data Sources**: Combines live web retrieval, structured financial APIs, and vector-indexed earnings reports for grounded synthesis.
*   **Key Stack**: Python, FastAPI, LangChain, Qdrant, Docker.

### 6. 🧰 [Agent & MCP Tooling Suite — ouss_mcp + ouss_agent](https://github.com/Oussama-nasri/ouss_mcp)
*A personal R&D lab for building and testing MCP servers and pure-Python AI agents — the foundation for every agentic capability I ship into production.*
*   **Engineering Highlights**:
    *   **ouss_mcp**: Custom MCP server exposing tools (web search, document retrieval, structured data access) consumed by agents via the MCP protocol.
    *   **ouss_agent**: Pure-Python agent built from first principles — tool dispatch loop, context window management, and multi-step reasoning without framework abstractions.
    *   **Protocol-Based Design**: Dependency injection via Python `Protocol` classes for graceful degradation across different tool backends.
*   **Key Stack**: Python, MCP SDK, FastAPI, Docker.
